## Section 1 — 产品定位（✅ approved）

### 1.1 产品名与定位

**思源学长**——给北京交通大学三校区（海淀本部 / 威海校区 / 雄安校区）学生用的、问什么答什么的"学长式"AI 助手。

定位关键词：**懂三校区差异**、**说人话**、**不编造**、**不带蒙骗倾向**。

### 1.2 目标用户（按优先级）

| 优先级 | 用户 | 核心需求 |
|--------|------|---------|
| **P1** | 准 / 新入学新生 | 入学前后所有疑问，避免被"假学长"带节奏 |
| **P2** | 跨校区在读老生 | 去别的校区办事 / 借资源 / 找差异 |
| **P3** | 普通服务问询用户（任意年级） | "我想办 X，去哪？找谁？要啥材料？" |

### 1.3 MVP scope（做什么）

- ✅ 三校区都做到一个水平面，**不分主次**
- ✅ 覆盖：入学手续、校区生活、服务流程、跨校区差异、防蒙骗信息核实
- ✅ 信息源透明 + 涉诈骗关键词标红

### 1.4 Non-scope（不做什么）

- ❌ 选课（教务系统耦合度太高，不在赛事范围）
- ❌ 实时通知（公告类信息走官方渠道）
- ❌ 心理咨询（伦理风险，超出 agent 能力）
- ❌ 学术辅导（不是 RAG 能解决的）
- ❌ 主动推送（赛题是"应用"不是"消息系统"）

### 1.5 防蒙骗机制（差异化卖点）

1. **信息源透明**：每条回答附引用源（知识库文档名 + 段落）
2. **引用原文**：关键事实（截止日期/费用/材料）必须从知识库原文摘录，不重写
3. **涉诈骗标记**：触发 8 类诈骗关键词时显式警告 + 给出官方渠道
4. **不编造**：知识库没有的事项明确说"我没查到，建议你去 [官方渠道]"

## Section 2 — 功能模块（✅ approved）

### 2.1 模块矩阵（M1–M8，M6 砍）

| ID | 模块名 | 优先级 | 说明 |
|----|--------|--------|------|
| M1 | 校区识别 + 路由 | P0 | 第一轮对话先识别用户所在校区（or 关心的校区） |
| M2 | 三校区知识问答（RAG） | P0 | 主功能，跨 4 个知识库检索 |
| M3 | 涉诈骗预警 | P0 | 关键词触发警告 + 引导官方渠道 |
| M4 | 跨校区差异查询 | P1 | "X 事情在三个校区分别怎么办" |
| M5 | 入学手册生成 | P1 | 按新生属性生成定制 PDF 入学指南 |
| ~~M6~~ | ~~跨校区资源对比~~ | ~~砍~~ | 与 M4 重叠，砍掉 |
| M7 | 思源学长名片 | P1 | 输出 H5 / 小程序卡片可分享（包含适当 CTA） |
| M8 | 入学倒计时 | P1 | 距离开学还有 N 天 + 今天该做啥 |

### 2.2 知识库结构（4 分区）

```
knowledge/
├── common/        # 三校区通用：校史 / 校规 / 学籍管理 / 学校 SSO / 全校性活动
├── haidian/       # 北京海淀本部专属
├── weihai/        # 威海校区专属
└── xiongan/       # 雄安校区专属
```

每个分区底下结构对齐（便于 M4 跨分区对比）：
- `入学/`：录取、报到、住宿、医保、银行卡、迎新流程
- `校园生活/`：宿舍、食堂、图书馆、运动设施、班车
- `教务服务/`：选课注册、成绩、转专业、辅修（信息向，不做操作）
- `行政服务/`：学生证、户籍、奖助贷、保险报销、出入校
- `周边/`：地理位置、交通、商圈、医院

### 2.3 主对话工作流（M3-M7 联动）

```
用户输入
  ↓
M1 校区识别（已识别 → 跳过 / 未识别 → 问一句）
  ↓
M3 诈骗关键词扫描（命中 → 警告 + 引导官方）
  ↓
意图分类：
  ├── 单校区问询 → M2 单分区 RAG
  ├── 跨校区对比 → M4 多分区 RAG + 表格对比输出
  ├── 入学手册 → M5 子流程
  └── 名片分享 → M7 卡片输出
  ↓
回答输出（附信息源 + 涉敏标记）
  ↓
M8 入学倒计时（仅新生 + 距开学 ≤ 30 天 时自动追加）
```

### 2.4 入学手册子流程（M5）

不是单次对话能完成的，需要**多轮收集属性**：
1. 你是哪个校区？
2. 报到日期？
3. 是否需要银行卡 / 医保 / 户籍迁移？
4. 是否走绿色通道（贫困生）？
5. 有无特殊餐饮 / 住宿需求？

→ 调用知识库 + 模板 → 输出**结构化 markdown 入学手册**，可下载为 PDF（Coze 内置插件支持）

### 2.5 涉诈骗关键词词典（8 类）

| 类别 | 示例触发词 |
|------|-----------|
| 金钱 | 转账 / 押金 / 充值 / 保证金 / 学费缴纳到个人账户 |
| 学习考证 | 包过 / 内部题库 / 保送 / 替考 / 论文代写 |
| 兼职就业 | 刷单 / 高薪兼职 / 内推保岗 / 押金兼职 |
| 情感 | 学长学姐"恋爱辅导" / 帮你撮合 |
| 冒充身份 | 自称辅导员私聊 / 校卡处发链接 |
| 租房物流 | 校外低价房（绕开官方） / 私收快递代收费 |
| 学校事务代办 | 代办学籍 / 代缴学费 / 代办奖学金 |
| 信息钓鱼 | 让你点链接填学号密码 / 奇怪的 QR 码 |

命中策略：**严格优先 recall over precision**——宁可多警告一次。

## Section 3 — 技术栈 + Coze 实现 + 测试（⏳ pending review）

### 3.1 技术栈

| 层 | 技术 | 备注 |
|----|------|------|
| LLM | 豆包 1.6 Pro | Coze 默认，赛事配套 |
| 平台 | 扣子 Coze Bot | 强制约束 |
| 知识库 | Coze 4 个独立 KB（common/haidian/weihai/xiongan） | 用 KB 名做路由依据 |
| 工作流 | Coze Workflow | M1/M3/M4/M5 用 Workflow，M2 用 Bot 直 RAG |
| 插件 | Coze 内置 OCR + PDF 生成 | M5 入学手册 PDF 导出 |
| 数据库 | Coze 内置 DB | 存用户校区 / 新生属性（长期记忆） |
| 长期记忆 | Coze 长期记忆 | 跨会话记住"用户是哪个校区的新生" |
| 触发器 | Coze 定时触发 | M8 入学倒计时（每天 8 点推一次给已识别新生，可选） |
| 前端 | 微信小程序 + H5 | Coze 一键发布渠道 |

### 3.2 Workflow 细节

#### Workflow A：主对话路由（包 M1 + M3 + 意图分类）

```
输入 user_msg
  ↓
[节点1] 调 LLM 提取：(校区, 意图类型, 是否涉诈骗关键词)
  ↓
[节点2] if 涉诈骗 → 返回警告模板 + 官方渠道，结束
  ↓
[节点3] 根据意图类型分支：
  - 单校区问询 → 调对应 KB
  - 跨校区对比 → 并行调多个 KB + 表格合并
  - 入学手册 → 启动 Workflow B
  - 其他 → 默认 fallback
```

#### Workflow B：入学手册多轮收集（M5）

Coze Workflow 支持多轮对话节点。
```
入口 → 问"你哪个校区" → 写入 DB → 问"报到日期" → 写入 DB → ...
→ 全部属性收集完 → 调 KB + 模板 → 生成 markdown → 调 PDF 插件 → 返回下载链接
```

### 3.4 测试策略

#### 50 标准问题集（自测）

按 5 维度 × 每维度 10 题：
1. **单校区基础问询**（10 题）：每个校区 3-4 题，覆盖常见入学问题
2. **跨校区对比**（10 题）：M4 模块的核心场景
3. **入学手册生成**（10 题）：M5 模块多轮交互
4. **诈骗预警**（10 题）：M3 触发 + 不误伤
5. **边界 case**（10 题）：知识库没有的事项 / 模糊问题 / 多意图

通过标准：**整体准确率 ≥ 80%**，M3 诈骗预警 recall ≥ 95%。

#### 真用户测试

- 5–10 个真实北交大学生（混合校区）
- 1 小时使用 + 10 分钟访谈
- 收集：完成任务率 / 信任度评分 / "我会推荐给学弟学妹吗" NPS

### 3.5 错误处理

| 场景 | 行为 |
|------|------|
| KB 没命中 | 明说"我没查到这个信息，建议去 [官方渠道]"，不编造 |
| LLM 工具调用失败 | 重试 1 次 → fallback 到纯对话模式 |
| 校区识别失败 | 主动问一句"你想问哪个校区的情况？" |
| 用户问超出 scope | 明说"我专注校园 X/Y/Z，这个问题我不懂" |

### 3.6 Claude Code vs Coze 分工

| 阶段 | 用什么 | 产出 |
|------|--------|------|
| 内容工厂 | Claude Code | 知识库 markdown 源材料 / 50 测试问题集 / 8 类诈骗词典 |
| 文档工厂 | Claude Code | 作品文档 / PPT 大纲 / 演示视频脚本 |
| 设计工厂 | Claude Code | persona prompt / workflow 流程图 / 工作流伪代码 |
| 组装线 | Coze 平台 | Bot 实际搭建 / KB 导入 / Workflow 节点连线 / 发布 |

**核心思路**：能在本地做的（结构化思考、文本生产、设计）全在 Claude Code 完成；Coze 平台只做"组装 + 发布"，避免在它身上花时间做不擅长的事。
