# CONTRIBUTING

## 角色分工
- **Lyric**：业务 owner，负责需求定义与最终审核
- **刀哥（AI）**：负责草案产出、结构化整理、PR 提交

## 提需求最小格式
```md
目标：
受众：
渠道：
约束：
截止时间：
验收标准：
```

## PR 标准
每个 PR 必须说明：
1. 为什么做（背景）
2. 做了什么（改动点）
3. 有什么风险（误导、侵权、敏感信息、品牌风险）
4. 如何验收（可量化指标）

## 内容安全边界
- 不提交任何账号密码/API Key
- 不提交未授权的私有资料
- 对外发布前必须人工复审

## 提交信息建议
- `feat: 新增 X 内容矩阵周计划模板`
- `docs: 更新 PR 审核清单`
- `fix: 修复复盘模板中的指标口径错误`
