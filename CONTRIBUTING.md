# CONTRIBUTING

## 角色分工
- **Lyric（Owner）**：定方向、定优先级、做最终审核
- **刀哥（Executor）**：拆任务、产出内容、提交 PR、维护文档

## 需求输入最小格式
```md
任务名：
目标：
交付物：
范围（含/不含）：
约束：
截止时间：
验收标准：
```

## PR 标准
每个 PR 至少说明：
1. 背景（为什么做）
2. 改动（做了什么）
3. 文件清单（改了哪些）
4. 风险与验收（怎么确保可用）

## 提交信息建议
- `feat: 新增项目周报模板`
- `docs: 更新知识沉淀目录说明`
- `fix: 修复任务模板字段错误`
